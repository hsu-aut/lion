Web Application Description Language

The WADL template uses a tabular approach for building a graph. First, a resource base path, and optionally a physical entity that provides that resource, have to be created. Afterwards, services can be added to existing resources. After having defined the services, requests and responses can be added as well.

The Web Application Description Language

Web Application Description Language (WADL) is W3C member submission defining a description language to model (RESTful) web services. It was originally developed as an XML representation. An ontology design pattern was created by the initial developers of Lions around 2019 and has been maintained by HSU AUT since.

With WADL, a available resources can be modelled together with the requests required to retrieve these resources via HTTP as well as the expected responses. Starting point of each WADL model is a base resource defining a base URL to which URLs of sub-resources can be appended. Every resource then contains a description of HTTP methods available to apply CRUD (create, read, update, delete) operations. Every method then contains a definition of the request that has to be made on that method. Requests basically act as containers for parameters which have to be submitted when making a request. Request parameters may be sent as header parameters, query parameters (i.e. with a variable name appended to a resource path), as so-called template parameters (i.e. as part of the path) or encoded with a certain media type using a representation element. Comparable to a request, possible responses of a method may also be modeled. A response contains the parameters that are sent together with the way they are encoded. Response parameters are encoded using a representation that defines the media type used for encoding the parameters (e.g. application/json)
